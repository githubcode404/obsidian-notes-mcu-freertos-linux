线程是调度的基本单位，进程则是资源分配的基本单位

**竞争条件**：
- 多线程相互竞争操作共享变量，导致输出存在不确定性
**临界区**：
- 访问共享资源的代码片段
**互斥**：
- 保证同一个时刻最多只有一个线程执行临界区代码
**同步**：
- 多个线程或者多个进程之间相互等待制约

实现互斥/同步
- 锁
- 信号量P/V

**Test and Set(&lock)**
- 原子操作，不会被打断
- 返回**锁的旧值**。如果是1，说明我没拿到锁；如果是0，说明我拿到了锁

**锁**
- 忙等待锁（**自旋锁**）：即等待锁可用时，始终**占用**CPU资源
- 无等待锁：获取不到锁，就进入等待队列，把CPU让给其他线程
- 当上下文切换开销较大时，使用自旋锁，等待几个周期的时间<上下文切换

**信号量**：
- p：sem--（可能阻塞）
- V：sem++（一定不会阻塞）
- 操作系统保证PV操作具有**原子性**
- ++后，如果sem≤0，则唤醒一个阻塞队列等待的进程/线程
-  - -后，如果sem<0，进程进入阻塞等待
-

**实现临界区的互斥访问**：
- sem == 1 无访问线程临界区
-  == 0 一个线程访问临界区
-  == -1 一个线程访问临界区，一个线程等待

实现**事件同步**：
- 一个线程（消费者）调用P，阻塞等待，另一个线程（生产者）在事件发生后调用V，唤醒前一个线程


**生产者-消费者**问题
- 生产，消费**同步**（这里的同步是相互依赖制约的意思，不是一起发生
- 使用两个同步信号量(full/empty)和一个互斥信号量（保证临界区）
- 同步信号量必须在互斥信号量前，否则会锁住
- empty表示空槽位，初始化为缓冲区大小
- full表示有数据的槽位，初始化为0
- mutex互斥信号量，初始化为1
```cpp
// 生产者
void producer() {
    while (1) {
        item = produce();           // 生产数据（在临界区外！）
        
        P(empty);   // ① 有空位吗？没有就阻塞（不抱锁！）
        P(mutex);   // ② 申请进临界区
        
        buffer[in] = item;          // ③ 放入缓冲区
        in = (in + 1) % N;
        
        V(mutex);   // ④ 退出临界区
        V(full);    // ⑤ 告诉消费者：有数据了！
    }
}

// 消费者
void consumer() {
    while (1) {
        P(full);    // ① 有数据吗？没有就阻塞
        P(mutex);   // ② 申请进临界区
        
        item = buffer[out];         // ③ 取出数据
        out = (out + 1) % N;
        
        V(mutex);   // ④ 退出临界区
        V(empty);   // ⑤ 告诉生产者：有空位了！
        
        consume(item);              // 消费数据（在临界区外！）
    }
}
```

**哲学家进餐问题**
- 死锁：每个人都拿一个方向的筷子，符合死锁的四个条件
- 方案1：添加全局锁，每次只有一个哲学家eat
- 方案2：添加局部锁，左右都有筷子时同时取左右两个筷子
- 方案3：奇数位置取左手，偶数位置取右手
- 以上方案都是在**打破死锁条件**


**读者-写者问题**
- 允许多个读者同时读
- 一个写者写时， 不允许读者读，也不允许其他写者写
- 

| 死锁            | 饿死                 |
| ------------- | ------------------ |
| **互相等待，循环卡住** | 一方**无限期被插队**，永远轮不到 |
| 所有相关进程都动不了    | **只有一方动不了**，其他很活跃  |
| 需要四条件同时满足     | 只需要调度不公平           |
- 公平版（queue保证处理顺序按照任务到的先后顺序
- 读者拿到queue后，占据rw，即刻放绳，保证多个读者并发读数据
- queue保证写者的位置不会被后到的读者占据
```cpp
semaphore queue = 1;   // 读写都先排队，保证先来后到，排队信号
semaphore rw = 1;      // 读写互斥
semaphore mutex = 1;   // 保护read_count
int read_count = 0;

// 读者（公平版）
void reader() {
    while (1) {
        P(queue);           // 排队！
        P(mutex);
        read_count++;
        if (read_count == 1) P(rw);
        V(mutex);
        V(queue);           // 拿到rw后放行后面的人（可能是读者或写者）
        
        // 读数据...
        
        P(mutex);
        read_count--;
        if (read_count == 0) V(rw);
        V(mutex);
    }
}

// 写者（公平版）
void writer() {
    while (1) {
        P(queue);           // 排队！
        P(rw);              // 独占
        
        // 写数据...
        
        V(rw);
        V(queue);           // 写完放行
    }
}
```
