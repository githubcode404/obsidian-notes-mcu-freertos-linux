|          | **栈（Stack）** | **堆（Heap）**              |
| :------- | :----------- | :----------------------- |
| **谁管理**  | 编译器自动管理      | 程序员手动管理（`malloc`/`free`） |
| **分配时机** | 函数调用时自动分配    | 运行时主动申请                  |
| **释放时机** | 函数返回时自动释放    | 程序员手动释放，或程序结束            |
| **生长方向** | 向低地址生长       | 向高地址生长                   |
| **速度**   | 极快（一条指令）     | 较慢（可能涉及系统调用）             |
| **大小**   | 有限（通常 1~8MB） | 较大（受限于虚拟内存）              |
| **碎片**   | 无（先进后出）      | 可能产生碎片                   |

**程序实例**
- **栈**

| 内容        | 说明                  |
| :-------- | :------------------ |
| **局部变量**  | `int b`, `char c` 等 |
| **函数参数**  | 传进来的值               |
| **返回地址**  | 函数执行完回到哪里           |
| **寄存器保存** | 调用前后需要恢复的寄存器值       |


```cpp
void foo(int a) {        // a 是参数，压栈
    int b = 10;          // b 是局部变量，压栈
    int arr[5];          // 局部数组，压栈
    bar();               // 调用 bar，返回地址压栈
}
```

- 堆

| 内容           | 说明                                |
| :----------- | :-------------------------------- |
| **动态分配的内存**  | `malloc`/`calloc`/`realloc` 返回的地址 |
| **动态数据结构**   | 链表节点、树节点、动态数组                     |
| **大对象**      | 栈放不下的（如大数组）                       |
| **跨函数存活的数据** | 函数返回后还要用的                         |

```cpp
void foo() {
    int *p = malloc(100);   // 堆上申请 100 字节
    // p 本身在栈上（局部指针变量）
    // p 指向的 100 字节在堆上
    free(p);                // 手动释放堆内存
}
```



- **关键联系：指针**
```cpp
void test() {
    int a = 10;              // a 在栈上
    int *p = malloc(4);      // p 在栈上，p 指向的内存 在堆上
    *p = 20;                 // 通过栈上的指针，操作堆上的数据
    
    // 函数返回时：
    // - a 销毁（栈自动释放）
    // - p 销毁（栈自动释放）→ 但堆上的 4 字节还在！
    // - 如果没有 free(p)，就内存泄漏了
    
    //free(p)
    //加上后堆上内存被释放，p变为野指针
    //p = NULL
    //把p指向NULL 
}
```
