
每个虚拟内存的内核地址，都是关联同一个物理内存的、
Linux用户空间
![[Pasted image 20260517213128.png]]
**malloc分配内存**
- brk（program break）把堆向高地址扩展，申请一块新区域分配内存
- mmap 从文件映射区分配一块内存

**malloc分配虚拟内存**
malloc每次分配内存会预分配更大的空间作为内存池
这部分空间保存了内存块的描述信息，比如有该内存块的大小（freertos提及）

**free释放后**
- brk方式：把内存放入内存池，下次申请无需重新分配内存。进程退出后堆内存释放
- mmap方式：释放后把虚拟内存归还操作系统
**优缺点**：
- brk：用户空间管理，速度快，但是会导致内存碎片（只有堆顶的空闲能回收）
- mmap：需进入内核空间管理，速度慢，但是每次释放后归还，申请则重新规划一块内存。容易产生内部碎片，这种内部碎片无法被检测，导致内存泄漏
```text
brk外部碎片产生原因：指针只能指向末尾空闲处
brk 的堆（一维连续）：
地址增长 → ─────────────────────────────────→
           ┌────┐┌────┐┌────┐┌────┐┌────┐
           │ A  ││ B  ││ C  ││ D  ││空闲 │
           └────┘└────┘└────┘└────┘└────┘
           
释放 B：     ┌────┐┌────┐┌────┐┌────┐┌────┐
            │ A  ││洞  ││ C  ││ D  ││空闲 │
            └────┘└────┘└────┘└────┘└────┘
                    ↑
                 这个洞被困住了！前面有A，后面有C/D，brk缩不回来

mmap 的分配（二维离散）：
           地址空间是一片大海
           
           ┌────┐              ┌────┐
           │ A  │              │ C  │
           └────┘              └────┘
           
    释放 A：munmap(A)
           
           [无]               ┌────┐
                              │ C  │
                              └────┘
           
           A 的位置回归"未定义"，不是"洞"，就是正常的未使用地址空间
```
因此，**二者结合使用**
- 几百k以下使用brk
- 几百k以上使用mmap
