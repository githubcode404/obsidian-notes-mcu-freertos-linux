
## 5
**mm_struct**
- 描述进程虚拟内存空间的内存描述符
- 内核线程mm_struct指向NULL
创建方式
1. 通过fork函数创建进程，copy_mm函数会把父进程的mm_struct拷贝到子进程，也就是当前创建的进程的虚拟内存空间是其父进程的拷贝、
2. vfork或者clone系统调用创建出的子进程，此时会把父进程的虚拟内存空间和相关页表直接赋值给子进程，此时，子进程和父进程共享虚拟内存空间，子进程变为线程

|                | 拷贝 (`fork`)        | 赋值/共享 (`vfork`/`clone`) |
| -------------- | ------------------ | ----------------------- |
| **mm\_struct** | 新建一个结构体，内容复制       | 直接指向父进程的结构体             |
| **物理地址**       | 同一个 `mm_struct` 对象 | 同一个 `mm_struct` 对象      |
| **页表**         | 复制一份新的             | 共用同一套                   |
| **物理页**        | 初始共享，写时复制分离        | 始终完全共享                  |
TIP：
- **是否共享地址空间**是线程和进程之间的本质区别

**take_size**：划分用户虚拟空间和内核虚拟空间（mm_struct中）
**vm**：virtual memory（虚拟内存）
**vm_area_struct**：描述虚拟内存空间区域VMA（代码段，数据段，BSS之类的区域）。包括（start，end），左闭右开

|          | `mm_struct`                 | `vm_area_struct` (VMA)    |
| -------- | --------------------------- | ------------------------- |
| **层级**   | 进程级，**整个地址空间的描述符**          | 区域级，**单个连续内存区域的描述符**      |
| **数量**   | 一个进程只有一个                    | 一个进程可以有**多个**（链表/红黑树组织）   |
| **类比**   | 整本《地图册》                     | 册子里的**每一页地图**             |
| **包含内容** | 所有 VMA 的集合、页表基址、RSS、虚拟内存统计等 | 一段连续虚拟地址的起止、权限、映射类型、文件偏移等 |
|          |                             |                           |
内核管理内存的最小单元是页

vm_page_prot：**内存页**的访问权限
vm_flags（众多标志位）：（一块）虚拟内存的访问权限

**关联内存映射中的映射关系**
匿名映射：虚拟内存映射到物理内存上
文件映射：需内存映射到文件上
**属性**：
- **anon_vma**：malloc申请较大内存时，使用mmap在文件映射和匿名映射区域创建VMA（匿名映射区域），使用anon_vma来表示这块区域
- **vm_file**：mmap进行文件映射时，关联被映射文件
- **vm_pgoff**：表示被关联文件内容在文件中的偏移
匿名映射指向物理内存，不指向文件，所以vm_file为NULL

**虚拟内存区域的相关操作**
vm_area_struct
- **vm_operations_struct**
   - 包含各类操作函数指针

**虚拟内存如何被组织**
- vm_area_struct结构体类型的双向链表
- 双向链表头指针存在内存描述符的mm_struct的mmap中，尾指针指向NULL
- 每个结构体内vm_mm都指向这一虚拟内存区域所属的虚拟内存空间mm_struct
内核中vm_area_struct：
- 双向链表：高效遍历
- 红黑树：高效查找(通过vm_rb连接到红黑树，根节点存储在mm_strcut中的mm_rb)
