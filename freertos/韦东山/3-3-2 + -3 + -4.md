
BL保存返回地址到LR →
进函数体时PUSH LR入栈 →
函数返回前POP LR出栈 →
BX LR返回到主调函数

LR(link register)：CPU中的链接**寄存器**
PC(program counter）：程序计数器，唯一决定"现在执行哪条指令"

**a函数中调用b函数**
BL指令
- 把调用b函数的程序语句的地址记录到LR，再跳转

进入b函数后
- 划分栈空间
- 保存当前LR(返回a函数的地址)（作为寄存器LR可能在嵌套函数调用关系中被覆盖，导致丢失，无法返回）
- 局部变量

POP LR：改变当前LR的值为栈内保存的值

BX LR
- 把LR存入PC

**-3局部变量分配内存**![[Pasted image 20260513112010.png]]
R0：寄存器
STR R0, {SP, #4}：把寄存器中的值存入栈中sp+4的位置，
+4：是+4bit，
注意栈是从高地址到低地址，为底部到顶部。所以栈底的地址比栈顶要高

若编译器优化，会在 局部变量放入寄存器 和 局部变量压入栈 之间做选择
不优化则局部变量压入栈中
