
句柄：指向TCB的指针

|          | **句柄（TaskHandle\_t）** | **TCB（TCB\_t）**       |
| -------- | --------------------- | --------------------- |
| **本质**   | 一个**指针**（`void *`）    | 一个**结构体**（包含大量字段）     |
| **大小**   | 4字节（32位）或 8字节（64位）    | 几十到上百字节               |
| **作用**   | **"门牌号"** — 用来找到任务    | **"房间本身"** — 存放任务所有信息 |
| **谁持有**  | 你的代码里保存               | FreeRTOS 内核管理         |
| **能做什么** | 传给 API 函数，让内核知道操作哪个任务 | 内核直接读写，调度时用           |

创建任务
1. 动态分配内存
```cpp
BaseType_t xTaskCreate( TaskFunction_t pxTaskCode, // 函数指针, 任务函数
                        const char * const pcName, // 任务的名字
                        const configSTACK_DEPTH_TYPE usStackDepth, // 栈大小,单位为word,10表示40字节
                        void * const pvParameters, // 调用任务函数时传入的参数
                        UBaseType_t uxPriority,    // 优先级
                        TaskHandle_t * const pxCreatedTask ); // 任务句柄, 以后使用它来操作这个任务
```
2. 静态分配内存
```cpp
TaskHandle_t xTaskCreateStatic ( 
    TaskFunction_t pxTaskCode,   // 函数指针, 任务函数
    const char * const pcName,   // 任务的名字
    const uint32_t ulStackDepth, // 栈大小,单位为word,10表示40字节
    void * const pvParameters,   // 调用任务函数时传入的参数
    UBaseType_t uxPriority,      // 优先级
    StackType_t * const puxStackBuffer, // 静态分配的栈，就是一个buffer
    StaticTask_t * const pxTaskBuffer // 静态分配的任务结构体的指针，用它来操作这个任务
);
```
TIP：
- 句柄（TaskHandle）和pxTaskBuffer都是指向TCB的指针
- 静态分配返回句柄，动态分配返回BaseType

我们通过链表找到任务的TCB，来控制任务

## 5-1-3
 为什么需要互斥访问：
- 当多个执行单元并发访问共享资源时，如果没有互斥保护，会导致**竞态条件（Race Condition）**
两个任务使用同一函数，且需要访问全局变量，此时需要互斥访问

void* param：指向任意类型的指针变量param

| 功能      | 是否需要句柄       |
| ------- | ------------ |
| 创建任务    | 不需要（传NULL也行） |
| 删除任务    | **需要**       |
| 挂起/恢复   | **需要**       |
| 改优先级    | **需要**       |
| 任务通知    | **需要**       |
| 查状态/栈用量 | **需要**       |

使用全局变量大概可以实现互斥
- 在执行功能函数前置0，执行后置1
- 只有1能进入功能函数
- 缺点：while循环导致当前任务执行后置1，又直接进入循环被置0，其他任务有几率始终无法执行，则全局变量可靠性大大降低
- 改进：在判断外加延时，使得每次置1后先切换另一个任务，避免被置0

5-4
**删除任务**：
- 频繁删除创建任务会造成内存碎片，影响内存分配
- 删除任务要做清楚工作

高优先级任务只要在就绪状态就不会主动让出CPU
- 高优先级任务只能主动离开，被挂起删除阻塞，
- vTaskDelay阻塞任务，使得低优先级任务得以执行，然后被抢断，回到高优先级任务，实现任务切换
- vTaskDelay是类似睡眠锁，释放cpu资源后阻塞
